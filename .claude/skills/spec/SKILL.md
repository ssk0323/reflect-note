---

ユーザーがリフレクションアプリの新機能や変更を提案してきた。仕様を Claude Code 主導でヒアリングし、Issue 化できるレベルまで具体化する。

# 手順

1. **spec-interviewer agent を起動する**
   - Agent ツールで `subagent_type: spec-interviewer` を指定
   - prompt には「ユーザーが何をしたいと言ったか」を 1〜2 文で渡す
   - 並行する別作業がない限り **foreground** で実行

2. **agent の結果を受け取り、ユーザーに最終仕様を見せる**
   - 仕様書 Markdown をそのまま会話に表示
   - 「これで Issue 化しますか？修正点はありますか？」と確認

3. **合意が取れたら次のステップを促す**
   - 「`/issue` で GitHub Issue にしますか？」
   - 仕様書 Markdown は次の `/issue` で再利用するため、会話の中に残す

# 注意

- 引数 `$ARGUMENTS` があれば、それを「ユーザーがやりたいこと」として agent に渡す
- 引数がなければ、まず「何の機能/変更について話しましょうか？」と聞いてから agent を起動
- 既に CLAUDE.md / 既存コードに書かれていることを聞き直さない（agent 側で読むよう指示済み）
